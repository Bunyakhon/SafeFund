คู่มือการติดตั้ง (Installation Guide)

1.ดาวน์โหลดโปรแกรม:
git clone https://github.com/Bunyakhon/SafeFund.git
cd SafeFund

2.ติดตั้ง Dependencies:
npm install

3.ตั้งค่า Environment:
สร้างไฟล์ .env ที่ root directory เพื่อกำหนดพอร์ต (ถ้ามี)
Code
PORT=5000
FRONTEND_PORT=5500
4.เริ่มใช้งานระบบ:
ใช้คำสั่งรันทั้ง Backend และ Frontend พร้อมกัน:
npm run dev
Frontend จะทำงานที่: FrontEnd SafeFund Run http://localhost:5500/
Backend จะทำงานที่: BackEnd SafeFund Run http://localhost:5000

คำอธิบายโครงงาน
1. ชื่อแอปพลิเคชัน
ระบบจัดการกองทุนหมู่บ้านและเงินกู้ SafeFund
SafeFund เป็นเว็บแอปพลิเคชันที่ออกแบบมาเพื่อบริหารจัดการกองทุนหมู่บ้านและเงินกู้ ระบบนี้ช่วยให้ผู้ใหญ่บ้านหรือผู้ดูแลสามารถจัดการเงินฝากสัจจะของสมาชิก ควบคุมการอนุมัติสัญญาเงินกู้ และติดตามการชำระคืนเงินกู้ได้อย่างเป็นระบบ
ระบบจัดการกองทุนหมู่บ้านและเงินกู้ ออกแบบมาเพื่อบริหารจัดการเงินฝากสัจจะของสมาชิก และควบคุมการอนุมัติรวมถึงการชำระคืนเงินกู้

รายการฟีเจอร์ที่ทำ
1.ระบบจัดการสมาชิก (Member Management):
เพิ่ม แก้ไข ลบ และดูรายชื่อสมาชิก
จัดเก็บข้อมูลพื้นฐาน เช่น ชื่อ ที่อยู่ และเบอร์โทรศัพท์

2.ระบบจัดการเงินกู้ (Loan Management):
สร้างสัญญาเงินกู้พร้อมกำหนดจำนวนเงิน อัตราดอกเบี้ย และระยะเวลา (เดือน)

3.ระบบคำนวณงวดชำระรายเดือนและดอกเบี้ยทั้งหมดโดยอัตโนมัติ
สร้างตารางการชำระเงิน (Payment Schedule) สำหรับทุกงวดทันทีที่สร้างสัญญา
บันทึกการยืนยันการชำระเงินในแต่ละงวด

4.ระบบปิดยอดอัตโนมัติ: 
เมื่อชำระครบทุกงวด 
สถานะสัญญาจะเปลี่ยนเป็น "ปิดยอดแล้ว" โดยอัตโนมัติ

5.ระบบจัดการเงินฝาก (Savings Management):
บันทึกรายการเงินฝากสัจจะรายวัน/รายเดือน
เรียกดูประวัติการฝากเงินแยกตามรายบุคคล

6.ระบบรายงาน (Reports):
รายงานสรุปสถานะเงินกู้คงค้างเพื่อติดตามหนี้
รายงานรวมยอดเงินฝากสะสมเพื่อใช้คำนวณปันผล

2. รายชื่อตาราง (4 ตาราง) พร้อมคอลัมน์หลัก
1)Member (ตารางสมาชิก)
-member_id (PK)
-member_name
-address
-phone
2)Loan (ตารางสัญญาเงินกู้)
-loan_id (PK)
-member_id (FK)
-loan_amount
-interest_rate
-status (เช่น Pending, Approved, Completed)
3)Payment (ตารางการชำระคืนเงินกู้ - Junction Table)
-payment_id (PK)
-loan_id (FK)
-amount
-payment_date
4)Saving (ตารางเงินฝากสัจจะ)
-saving_id (PK)
-member_id (FK)
-deposit_amount
-deposit_date

3. ความสัมพันธ์ระหว่างตาราง (Relationships)
One-to-Many: Member 1 : N Saving (สมาชิกหนึ่งคนฝากเงินสะสมได้หลายครั้ง)
One-to-Many: Member 1 : N Loan (สมาชิกหนึ่งคนกู้ยืมได้หลายสัญญา)
Many-to-Many: ในที่นี้ Payment ทำหน้าที่เป็น Junction Table เชื่อมระหว่าง Member และ Loan (ผ่าน loan_id) เพื่อจัดการยอดชำระที่แบ่งเป็นงวดๆ ทำให้หนึ่งสัญญาเงินกู้มีรายการชำระเงินย้อนหลังได้หลายรายการ และสมาชิกสามารถติดตามยอดที่จ่ายไปแล้วในแต่ละสัญญาได้อย่างแม่นยำ

4. คำอธิบายรายงานที่วางแผนจะทำ (2 รายงาน)
รายงานที่ 1: รายงานสรุปสถานะเงินกู้คงค้าง
แสดงรายชื่อสมาชิกที่ยังมีสัญญาเงินกู้ค้างชำระ พร้อมยอดหนี้รวมที่ต้องเรียกเก็บ
รายงานที่ 2: รายงานเงินฝากสะสมรายปี
แสดงยอดรวมเงินฝากสัจจะของสมาชิกแต่ละคนเพื่อใช้คำนวณเงินปันผลประจำปี
