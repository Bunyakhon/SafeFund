# SafeFund
1. ชื่อแอปพลิเคชัน
SafeFund
ระบบจัดการกองทุนหมู่บ้านและเงินกู้ ออกแบบมาเพื่อบริหารจัดการเงินฝากสัจจะของสมาชิก และควบคุมการอนุมัติรวมถึงการชำระคืนเงินกู้

2. รายชื่อตาราง (4 ตาราง) พร้อมคอลัมน์หลัก
1)Member (ตารางสมาชิก)
-member_id (PK)
-member_name
-address
-phone
2)Loan (ตารางสัญญาเงินกู้)
-loan_id (PK)
-member_id (FK)
-loan_amount
-interest_rate
-status (เช่น Pending, Approved, Completed)
3)Payment (ตารางการชำระคืนเงินกู้ - Junction Table)
-payment_id (PK)
-loan_id (FK)
-amount
-payment_date
4)Saving (ตารางเงินฝากสัจจะ)
-saving_id (PK)
-member_id (FK)
-deposit_amount
-deposit_date

3. ความสัมพันธ์ระหว่างตาราง (Relationships)
One-to-Many: Member 1 : N Saving (สมาชิกหนึ่งคนฝากเงินสะสมได้หลายครั้ง)
One-to-Many: Member 1 : N Loan (สมาชิกหนึ่งคนกู้ยืมได้หลายสัญญา)
Many-to-Many: ในที่นี้ Payment ทำหน้าที่เป็น Junction Table เชื่อมระหว่าง Member และ Loan (ผ่าน loan_id) เพื่อจัดการยอดชำระที่แบ่งเป็นงวดๆ ทำให้หนึ่งสัญญาเงินกู้มีรายการชำระเงินย้อนหลังได้หลายรายการ และสมาชิกสามารถติดตามยอดที่จ่ายไปแล้วในแต่ละสัญญาได้อย่างแม่นยำ

4. คำอธิบายรายงานที่วางแผนจะทำ (2 รายงาน)
รายงานที่ 1: รายงานสรุปสถานะเงินกู้คงค้าง
แสดงรายชื่อสมาชิกที่ยังมีสัญญาเงินกู้ค้างชำระ พร้อมยอดหนี้รวมที่ต้องเรียกเก็บ
รายงานที่ 2: รายงานเงินฝากสะสมรายปี
แสดงยอดรวมเงินฝากสัจจะของสมาชิกแต่ละคนเพื่อใช้คำนวณเงินปันผลประจำปี
